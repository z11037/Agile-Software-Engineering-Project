# Agile Software Engineering Project Plan (Week 2–Week 6)

## Project Goal
Design and prototype a software product that helps students **learn English effectively** through:

- Vocabulary review
- Quizzes and exercises
- Learning progress tracking
- Personalized learning suggestions

---

# Team Roles (7 People)

**Product / Management**

- **Product Owner (1)** – defines requirements, manages backlog
- **Scrum Master (1)** – facilitates meetings and removes blockers

**Development**

- **Frontend Developers (2)** – UI, quiz interface, dashboards
- **Backend Developers (2)** – API, database, learning logic

**Quality**

- **QA / Testing (1)** – testing, bug tracking, validation

---

# Week 2 – Project Initiation & Requirement Elicitation

## Goals
Understand the problem and define the **Minimum Viable Product (MVP)**.

## Tasks

### 1. User Research
- Interview students
- Identify common English learning problems

### 2. Define User Stories

Example:

- As a student, I want to **review vocabulary daily**
- As a student, I want to **take quizzes**
- As a student, I want to **track my learning progress**
- As a student, I want **reminders to study**

### 3. Create Product Backlog

| Priority | Feature |
|--------|--------|
| High | Vocabulary review system |
| High | Quiz system |
| High | User progress tracking |
| Medium | Daily learning reminder |
| Medium | Learning statistics dashboard |
| Low | Personalized recommendations |

### 4. Define MVP

MVP Features:

- Word review
- Quiz system
- Basic progress tracking

### 5. Architecture Planning

Decide:

- Frontend: Web / Mobile
- Backend: API Server
- Database: Vocabulary + user data

---

# Week 3 – Sprint 1 (Core System Development)

## Sprint Goal
Build the **basic learning system**.

## Tasks

### Frontend
- Login / user interface
- Word review interface
- Quiz page

### Backend
- User account system
- Vocabulary database
- Quiz API

### QA
- Write test cases
- Test quiz functionality

## Deliverable

Working prototype with:

- User login
- Vocabulary review
- Simple quiz

---

# Week 4 – Sprint 1 Review & Sprint 2 Planning

## Sprint Review
Demonstrate:

- Vocabulary review feature
- Quiz system

Collect feedback:

- Is the UI easy to use?
- Is the quiz difficulty appropriate?

## Sprint Retrospective

Discuss:

- What went well
- What needs improvement

## Sprint 2 Planning

Add new features:

- Progress tracking
- Learning statistics
- Improved quiz system

---

# Week 5 – Sprint 2 (Feature Expansion)

## Sprint Goal
Improve learning experience and add analytics.

## Tasks

### Frontend
- Learning progress dashboard
- Statistics visualization

### Backend
- Track quiz scores
- Store learning history
- Progress calculation

### QA
- Test progress tracking
- Test dashboard data

## Deliverable

System with:

- Quiz history
- Learning statistics
- Progress tracking

---

# Week 6 – Final Integration & Demonstration

## Tasks

- Bug fixing
- UI improvements
- Final testing
- Prepare final presentation

## Final Demo Flow

1. Student logs in
2. Reviews vocabulary
3. Takes a quiz
4. System tracks learning results
5. Dashboard shows learning progress
